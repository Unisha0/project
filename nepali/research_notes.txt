1. TOKENIZATION — HOW TEXT BECOMES NUMBERS
================================================================================

Models don't read text — they read numbers. Tokenization splits text into
smaller units (tokens) and maps each to a numerical ID.

Three Levels of Tokenization:
-----------------------------

a) Word-level Tokenization
   Example (Nepali): ["हाम्रो", "टोलमा", "बत्ती", "छैन"]
   Pros:  Intuitive, preserves word boundaries
   Cons:  Huge vocabulary size, can't handle unseen/rare words (OOV problem)

b) Character-level Tokenization
   Example (Nepali): ["ह", "ा", "म", "्", "र", "ो"]
   Pros:  Very small vocabulary, handles any word
   Cons:  Very long sequences, loses word-level semantics

c) Subword Tokenization (WordPiece)
   Example (Nepali): ["हाम्", "##रो", "टोल", "##मा", "बत्ती"]
   Pros:  Best balance — small vocab, handles unseen words via subword merging
   Cons:  Needs a pre-trained tokenizer for the target language

   This is the standard approach used by DistilBERT multilingual and BERT-family models.

---

2. THE TOKENIZER PROBLEM FOR NEPALI
================================================================================

Your English-only DistilBERT model cannot handle Devanagari:

   Input:  "हाम्रो टोलमा बत्ती छैन"
   Tokens: ["[UNK]", "[UNK]", "[UNK]", "[UNK]"]

Solution: Use **distilbert-base-multilingual-cased** tokenizer.

Example:

   Input: "हाम्रो टोलमा बत्ती छैन"
   Tokens: ["हाम्", "##रो", "टोल", "##मा", "बत्ती", "छैन"]

Meaningful subwords, no [UNK] problem.

Comparison of Multilingual Tokenizers:
--------------------------------------
+---------------------------------------+------------------+----------------+--------+
| Model                                 | Tokenizer        | Nepali Support | Params |
+---------------------------------------+------------------+----------------+--------+
| distilbert-base-multilingual-cased    | WordPiece 110K   | Yes            | 66M    |
| bert-base-multilingual-cased          | WordPiece 110K   | Yes            | 110M   |
| xlm-roberta-base                      | SentencePiece 250K| Best          | 125M   |

3. PRE-TRAINED LANGUAGE MODELS (PLMs)
================================================================================

What is a Pre-trained Language Model?
-------------------------------------
A model trained on massive amounts of unlabeled text using self-supervised objectives:

- Masked Language Modeling (MLM): Mask random tokens, predict them from context.
  Example: "हाम्रो टोलमा [MASK] छैन" → predict "बत्ती"

- These models learn grammar, semantics, and domain knowledge transferable to your task.

Types of Models:
----------------
a) Encoder-only (BERT / DistilBERT):
   - Best for classification, NER, sentiment analysis
   - Includes: DistilBERT, BERT, RoBERTa
   - This is what you need for complaint classification

b) Decoder-only (GPT-family):
   - Best for text generation

c) Encoder-Decoder (T5/BART-family):
   - Best for translation, summarization

4. KEY CONCEPTS BEFORE TRAINING
================================================================================

a) Fine-tuning
   - Take a pre-trained DistilBERT multilingual
   - Add a classification head (linear layer: 768 → 4 classes)
   - Train on your labeled dataset
   - Typical config: 3-5 epochs, learning rate 2e-5, batch size 16-32

b) Embeddings
   - Each token → 768-dimensional vector
   - Contextual embeddings differentiate word meaning per sentence

c) Attention Mechanism
   - Multi-head self-attention focuses on relevant words in input
   - For "बत्ती आउँदैन", attention highlights "बत्ती"

d) Transfer Learning
   - Pre-training on 100+ languages → fine-tuning on Nepali complaints
   - Reduces data needed for good performance

e) Label Encoding
   - electricity → 0
   - water       → 1
   - road        → 2
   - garbage     → 3

f) Loss Function: Cross-Entropy Loss for multi-class classification
g) Optimizer: AdamW, lr=2e-5
h) Padding & Truncation: max_length=128, attention_mask differentiates padding

5. PREPROCESSING PIPELINE
================================================================================

Raw CSV → Merge → Clean → Tokenize → Encode → DataLoader → Model

Steps:
------
1. Merge CSVs: electricity, water, road, garbage → one DataFrame
2. Label encode categories {electricity:0, water:1, road:2, garbage:3}
3. Light text cleaning: whitespace, punctuation, stray English removed
4. Tokenization: DistilBERT multilingual tokenizer → token IDs & attention masks
5. Dataset creation: train/val/test splits
6. Dataloader: batch_size=16–32, shuffle training data
7. Training: fine-tune model, save best checkpoint based on val loss

8. TRAINING STRATEGY
================================================================================

----------------
